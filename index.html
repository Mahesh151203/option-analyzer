⚠ Daily Session Lifecycle
Angel One Smart API JWT tokens are valid for one trading day only, expiring at 3:30 PM IST.

Each morning you must:
1. Open this app → Session / Login tab
2. Enter credentials → Generate Session
3. JWT is stored in memory for the day
4. Switch to Analyzer → Fetch Live Data

The refresh token can extend sessions up to 7 days but resets on each new login.
Never share your API Key or TOTP secret.
Angel One Smart API — Setup Guide & Requirements

Step 1 — Create Smart API App
Visit smartapi.angelone.in → Login with your Angel One trading account → Create a new app → Copy your API Key.

Step 2 — Enable TOTP (2FA)
In Smart API console → enable TOTP → scan the QR with Google Authenticator / Authy → save the base32 secret shown below the QR. You need this for automated login.

Step 3 — Install SDK (for backend / production)
pip install smartapi-python  or  npm install smartapi-javascript
For browser use, this app calls the REST API directly. See CORS note below.

Session Expiry Details
• JWT valid until 3:30 PM IST (end of trading day) — must re-login each morning
• Refresh token extends up to 7 days — resets on new login
• This app stores tokens in memory only — lost on page refresh
• For production: use a Node/Python backend that auto-refreshes tokens

Rate Limits
• Option chain API: 1 req/second
• Quote API: 10 req/second
• Login: 3 attempts/hour

⚠ CORS Limitation: Angel One Smart API blocks direct browser-to-API calls. For reliable live data in a browser, set up a thin backend proxy:
Node/Express or Python FastAPI running on localhost that forwards requests with proper headers.
